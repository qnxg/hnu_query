## 提交 Issue

提交 Issue 是最简单直接的贡献方法。你可以在 Issue 中提出希望 HNU Query 新增的功能（新增的功能大概率也会同步上线到微生活小程序中），或是指出 HNU Query 存在的问题。

如果你是微生活小程序的用户，当你观察到微生活无法获取到一些数据，或是获取到的数据存在错误时，这往往意味着 HNU Query 出现了一些问题，你可以根据 `docs/test.md` 中 “实际请求测试” 部分的内容，使用你自己的账号信息对 HNU Query 进行测试，如果测试失败则说明确实存在问题，请将测试失败的输出附加到 Issue 的描述中。

## 提交 PR

你可以查看目前已有的 Issue，或是根据你自己的想法提交 PR。

PR 不一定意味着一定要贡献实在的代码，你还可以贡献如下的内容：

* 增加文档注释，比如为一些结构体字段添加更多可能的示例，我们现在的项目比较缺少这个。
* 补充测试，TODO

提交 PR 前请确保你的代码经过了 `cargo fmt` 格式化，并通过了 `cargo clippy` 检查。

本项目使用 `AGPL-3.0`。提交贡献即表示你同意以该协议授权你的贡献代码。
